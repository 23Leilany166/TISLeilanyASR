Tabla Clientes

Propósito

La tabla Clientes es fundamental para almacenar la información de los clientes que interactúan con la constructora. Permite gestionar la identidad y los datos de contacto de cada cliente, asegurando una comunicación efectiva y un acceso seguro a sus cuentas.

Campos

  • ID_Cliente: Clave primaria única que identifica a cada cliente.
  • Nombre: Nombre completo del cliente.
  • Correo: Correo electrónico único y obligatorio, permite la autenticación y comunicación efectiva.
  • Teléfono: Campo obligatorio que facilita el contacto directo con el cliente.
  • Dirección: Campo que registra la dirección del cliente, esencial para la gestión de proyectos.
  • Sexo: Sexo del cliente.
  • FechaNacimiento: Fecha de nacimiento del cliente.
  • Edad: Edad calculada del cliente.

Tabla Trabajadores

Propósito

La tabla Trabajadores almacena la información de los empleados que laboran en la constructora, facilitando la gestión del personal y la asignación de proyectos.

Campos

  • ID_Trabajador: Clave primaria única que identifica a cada trabajador.
  • Nombre: Nombre completo del trabajador.
  • Correo: Correo electrónico único y obligatorio.
  • Teléfono: Contacto directo del trabajador.
  • Dirección: Información de la residencia del trabajador.
  • Sexo: Sexo del trabajador.
  • FechaNacimiento: Fecha de nacimiento del trabajador.
  • Puesto: Cargo o posición del trabajador en la empresa.

Tabla Proveedores

Propósito

La tabla Proveedores permite gestionar la información de los proveedores que suministran materiales y servicios a la constructora.

Campos

  • ID_Proveedor: Clave primaria única que identifica a cada proveedor.
  • Nombre: Nombre del proveedor.
  • Correo: Correo electrónico del proveedor.
  • Teléfono: Contacto directo del proveedor.
  • Dirección: Información de la ubicación del proveedor.
  • Sexo: Sexo del contacto principal del proveedor.
  • FechaNacimiento: Fecha de nacimiento del contacto (si aplica).
  • Estado: Indica si el proveedor está activo o bloqueado.

Tabla Planos

Propósito

La tabla Planos almacena los detalles de los planos de construcción asociados a proyectos específicos, facilitando su gestión y seguimiento.

Campos

  • ID_Plano: Clave primaria única que identifica cada plano.
  • Código: Código único del plano.
  • Calle: Calle del proyecto asociado.
  • Número: Número de la propiedad del proyecto.
  • ProyectoAsociado: Clave foránea que relaciona el plano con un proyecto específico.
  • Encargado: Nombre del encargado del plano.

Código SQL para crear tablas


        CREATE DATABASE ConstructoraBD;
        USE ConstructoraBD;

        CREATE TABLE Clientes (
            ID_Cliente INT PRIMARY KEY IDENTITY,
            Nombre NVARCHAR(100) NOT NULL,
            Correo NVARCHAR(100) UNIQUE NOT NULL,
            Teléfono NVARCHAR(15) NOT NULL,
            Dirección NVARCHAR(255) NOT NULL,
            Sexo CHAR(1),
            FechaNacimiento DATE,
            Edad INT
        );

        CREATE TABLE Trabajadores (
            ID_Trabajador INT PRIMARY KEY IDENTITY,
            Nombre NVARCHAR(100) NOT NULL,
            Correo NVARCHAR(100) UNIQUE NOT NULL,
            Teléfono NVARCHAR(15) NOT NULL,
            Dirección NVARCHAR(255) NOT NULL,
            Sexo CHAR(1),
            FechaNacimiento DATE,
            Puesto NVARCHAR(100)
        );

        CREATE TABLE Proveedores (
            ID_Proveedor INT PRIMARY KEY IDENTITY,
            Nombre NVARCHAR(100) NOT NULL,
            Correo NVARCHAR(100) UNIQUE NOT NULL,
            Teléfono NVARCHAR(15) NOT NULL,
            Dirección NVARCHAR(255) NOT NULL,
            Sexo CHAR(1),
            FechaNacimiento DATE,
            Estado NVARCHAR(10) CHECK (Estado IN ('Activo', 'Bloqueado'))
        );

        CREATE TABLE Planos (
            ID_Plano INT PRIMARY KEY IDENTITY,
            Código NVARCHAR(50) NOT NULL,
            Calle NVARCHAR(100) NOT NULL,
            Número NVARCHAR(10) NOT NULL,
            ProyectoAsociado INT,
            Encargado NVARCHAR(100),
            FOREIGN KEY (ProyectoAsociado) REFERENCES Proyectos(ID_Proyecto)
        );
